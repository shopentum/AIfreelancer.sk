# Terminológia — Greenfield MVP DNA

## Fázy

| Termín | Význam |
|--------|--------|
| **MVP delivery** | Termín-bound slice: demo / pilot / first release |
| **Phase 2** | Metodika, scale, analytics depth, hardening — po prvom dôkaze |
| **Discovery / spike** | Rozhodnutie pred buildom; výstup = rozhodnutie + DATA GAP |

## Scope a kvalita

| Termín | Význam |
|--------|--------|
| **Slice** | Najmenší rez, ktorý dokazuje hodnotu |
| **Shape > completeness** | Konzistentná kostra pred 100 % funkcií |
| **Delivery language** | Čo, kedy, AC — bez vnútornej metodiky |
| **STOP CONDITIONS** | Zastav a zúž — pozri SKILL.md |

## Meranie (generic)

| Termín | Význam |
|--------|--------|
| **Usage / interaction** | Správanie v produkte (klik, dokončenie, adopcia nástroja) |
| **Outcome / performance** | Výsledok po akcii (konverzia, čas, revenue — ak existuje) |
| **Eligible báza** | Menovateľ len tam, kde má feature zmysel |
| **DATA GAP** | Pomenovaná medzera; kto rozhodne; blokuje alebo nie |

## AI v MVP

| Termín | Význam |
|--------|--------|
| **HITL** | Človek potvrdí pred kritickou akciou |
| **Orchestrácia** | Spojiť existujúce API/moduly; nie nový monolit |
